Intent: Add MCP Streamable HTTP transport and A2A Agent Card support

## What shipped

### 1. MCP Streamable HTTP transport (commit 3f41a3e)
- `POST/GET/DELETE /mcp` endpoint using `WebStandardStreamableHTTPServerTransport`
- Shared tool registration extracted into `packages/mcp-server/src/tools.ts`
- Per-session McpServer instances with 30-min timeout cleanup
- Auth via `Authorization: Bearer sk_live_...` header
- `strale_search` works without auth; all other tools require auth
- CORS headers for browser-based clients
- Stdio transport unchanged (verified still works)
- README updated with remote connection config
- Dockerfile build order changed: mcp-server first, then api

### 2. A2A Agent Card + task endpoint (commit 87d7d51)
- `GET /.well-known/agent-card.json` — dynamic Agent Card (233 skills from DB, 5-min cache)
- `POST /a2a` — JSON-RPC endpoint supporting `message/send` and `tasks/get`
- Agent Card aliases: `/.well-known/agent.json`, `/agent.json`
- `Link: </.well-known/agent-card.json>; rel="agent-card"` header on all responses
- Thin proxy pattern: A2A tasks call POST /v1/do internally
- Cache-Control: public, max-age=300 on Agent Card

## Production URLs
- MCP: `https://api.strale.io/mcp`
- A2A Agent Card: `https://api.strale.io/.well-known/agent-card.json`
- A2A Task endpoint: `https://api.strale.io/a2a`

- A2A: Agent Card (233 skills), message/send (vat-format-validate), tasks/get, error cases
